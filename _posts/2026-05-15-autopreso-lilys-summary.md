## 한 줄 요약
autopreso는 **사용자가 말하는 내용을 실시간으로 화이트보드에 시각화해 주는 발표 보조 도구**이고,
이번 리포트의 핵심은 **발표 자료 제작과 아이디어 정리를 손 대신 음성 + AI 에이전트 흐름으로 바꾸려는 시도**라는 점이었습니다.

## 왜 이 내용을 남기나
발표 자료를 만들 때 시간이 많이 드는 부분은 꼭 슬라이드 디자인 자체만은 아닙니다.

- 생각을 구조화하는 일
- 말할 내용을 시각적으로 옮기는 일
- 발표 중 흐름에 맞춰 자료를 조정하는 일
- 아이디어를 빠르게 그려 보는 일

이 리포트는 그런 작업을 **“직접 그리는 일”에서 “말로 지시하는 일”로 바꾸려는 도구**를 소개합니다.
특히 화이트보드 기반 발표나 아이디어 설명에 관심이 있다면 꽤 흥미로운 방향입니다.

## autopreso는 어떤 도구인가
리포트가 설명하는 autopreso는,
**사용자의 음성을 듣고 그것을 텍스트로 바꾼 뒤, AI가 그 내용을 이해해서 화이트보드에 실시간으로 반영하는 도구**입니다.

쉽게 말하면:
- 내가 말한다
- 음성 인식이 텍스트로 바꾼다
- AI 에이전트가 내용을 이해한다
- 화이트보드에 그림이나 구조를 그린다

이 흐름입니다.

즉 발표 자료를 미리 완성해 두는 방식만이 아니라,
**말하면서 설명 구조를 시각적으로 함께 만들어 가는 방식**에 더 가깝습니다.

## 이 리포트가 말하는 핵심
리포트 내용상 autopreso의 핵심 가치는 크게 네 가지로 정리할 수 있습니다.

### 1. 발표를 손보다 말 중심으로 바꾼다
기존 발표 준비는 대개:
- 슬라이드를 직접 만들고
- 요소를 옮기고
- 정렬하고
- 시각 구조를 손으로 조정하는 흐름입니다.

autopreso는 여기서 한 걸음 바꿉니다.
핵심은 **“내가 말한 내용을 기반으로 AI가 화이트보드를 정리한다”**는 점입니다.

그래서 발표자는:
- 마우스/키보드 조작보다
- 설명 내용과 흐름 자체에 더 집중할 수 있습니다.

### 2. 실시간 시각화가 가능하다
리포트는 autopreso를 단순 메모 앱이 아니라,
**실시간으로 계속 업데이트되는 화이트보드 도구**로 설명합니다.

말하는 도중에도:
- 개념 연결
- 구조 정리
- 설명용 도식
- 강조 포인트

같은 것이 계속 반영될 수 있다는 뜻입니다.

### 3. 원하는 모델을 가져다 쓸 수 있다
리포트에서 강조한 또 하나의 포인트는 **Bring your own model**입니다.

예를 들면:
- OpenAI API 키를 사용할 수 있고
- Codex 계열 환경을 붙일 수 있고
- Ollama 같은 로컬 모델도 활용할 수 있습니다.

즉 이 도구 자체가 특정 SaaS에 완전히 묶인 구조라기보다,
**AI 에이전트 층을 비교적 유연하게 선택할 수 있는 방향**으로 보입니다.

### 4. 로컬 실행 가능성이 중요하다
리포트에서는 Moonshine, Ollama 같은 구성도 언급합니다.
이 흐름이 중요한 이유는,
**네트워크 의존도를 낮추고 로컬 환경에서 돌릴 수 있는 가능성**을 준다는 점입니다.

발표나 아이디어 정리 도구는 민감한 내용이 오갈 수도 있기 때문에,
이런 로컬 실행 가능성은 꽤 큰 장점으로 느껴집니다.

## autopreso는 어떻게 작동하나
리포트 기준으로 보면 작동 구조는 비교적 단순합니다.

### 1단계: 음성 인식
사용자의 말을 듣고 텍스트로 바꿉니다.

리포트에서는:
- Moonshine
- OpenAI Realtime

같은 음성 입력 경로가 언급됩니다.

### 2단계: AI 에이전트 해석
텍스트로 바뀐 내용을 AI가 읽고,
그 내용을 어떤 구조와 시각 표현으로 옮길지 판단합니다.

### 3단계: 화이트보드 반영
그 결과를 화이트보드에 실시간으로 반영합니다.

즉 autopreso는 결국:
**음성 입력 → 의미 해석 → 시각 출력**
이라는 3단 구조로 이해할 수 있습니다.

## autopreso의 특별한 점
### 1. Hands free
리포트에서 가장 직관적으로 와 닿는 표현은 **Hands free**입니다.

발표 중에:
- 슬라이드를 따로 만지기 어렵고
- 설명 흐름이 끊기기 쉽고
- 생각이 나면 즉시 구조화하고 싶을 때가 있는데

autopreso는 그런 순간에,
**손 대신 말로 제어하는 발표 보조 도구**라는 포지션을 갖습니다.

### 2. 무료 도구 + 외부 모델 결합 가능
리포트 설명상 autopreso 자체는 무료 도구이고,
실제 비용은 사용자가 붙이는 모델/서비스에 따라 달라질 수 있습니다.

이 구조는 꽤 현실적입니다.
툴은 무료로 열어두고,
사용자는 필요에 따라:
- 로컬 모델을 붙이거나
- OpenAI를 붙이거나
- Codex 환경을 쓰는 식으로 선택하게 됩니다.

### 3. 발표 준비보다 아이디어 스케치에도 잘 맞아 보인다
리포트는 발표 도구로 설명하지만,
실제로는 발표만이 아니라:
- 개념 설명
- 브레인스토밍
- 구조 초안 만들기
- 교육용 설명 흐름 정리

같은 용도에도 잘 맞아 보입니다.

## 설치와 실행 흐름
리포트에서 정리한 실행 방식은 대략 아래와 같습니다.

### 가장 간단한 실행
```bash
npx autopreso
```

설치 없이 바로 실행해 보는 흐름입니다.

### 전역 설치
```bash
npm install -g autopreso
autopreso
```

조금 더 안정적으로 계속 쓸 생각이면 이 방식이 더 자연스럽습니다.

### 개발자 방식
GitHub 저장소를 받아 직접 실행하는 흐름도 가능합니다.
리포트는 개발자라면 소스 코드 기반 설치도 가능하다고 설명합니다.

## 실행 후 사용 흐름
프로그램이 실행되면 브라우저가 열리고,
그 안에서 대략 다음을 설정하게 됩니다.

- 마이크
- 음성 인식 엔진
- AI 모델
- 발표 시작 전 기본 지시사항

그리고 **발표 시작**을 누르면,
말하는 내용에 맞춰 화이트보드가 업데이트되는 방식입니다.

리포트에서 인상적이었던 점은,
**미리 화이트보드에 조금 그려 두면 AI가 더 잘 이해할 수 있다**는 설명이었습니다.
이건 완전 자동 생성이라기보다,
사람이 약간의 구조를 주고 AI가 그 위에서 확장하는 방식으로 읽혔습니다.

## 내부 작동 방식에서 눈에 띄는 점
### 준비 모드와 실시간 모드
리포트는 두 가지 모드를 설명합니다.

- 준비 모드(staging): 발표 전에 직접 기본 구조를 그려두는 모드
- 실시간 모드(live): 발표 중 음성에 맞춰 자동으로 반영되는 모드

이 구분은 꽤 중요합니다.
완전 무(無)에서 시작하는 것보다,
**준비 모드에서 맥락을 심고 실시간 모드에서 확장하는 흐름**이 더 현실적이기 때문입니다.

### 설정 저장
리포트에 따르면 설정은 아래 파일에 저장됩니다.

```bash
~/.config/autopreso/settings.json
```

즉:
- 선택한 모델
- API 키
- 음성 인식 엔진
- 기타 기본 설정

이 재실행 후에도 유지됩니다.

### Warmup loop
발표 시작 전에 AI가 미리 준비하는 워밍업 흐름이 있다는 설명도 흥미로웠습니다.
이건 실시간 시스템에서 꽤 중요합니다.
사용자가 첫 문장을 말했을 때 지연이 크면 전체 경험이 나빠지기 때문입니다.

## 명령어와 설정 포인트
리포트에서 언급된 기본 명령어는 아래 정도입니다.

```bash
autopreso
autopreso -h
autopreso --no-open
```

설정 면에서는:
- 설정 파일 직접 관리
- 에이전트 지시사항 입력
- 비용 표시
- 초기 실행 시 환경 자동 감지

같은 포인트가 눈에 띄었습니다.

특히 리포트는 초기 자동 감지 우선순위도 설명합니다.
예를 들면:
- Codex CLI
- OLLAMA_MODEL
- OPENAI_API_KEY

같은 값을 기준으로 어느 경로를 기본값으로 잡을지 판단하는 식입니다.

## 환경 변수 메모
리포트에서 언급된 환경 변수는 다음과 같습니다.

- `PORT`
- `OPENAI_API_KEY`
- `OPENAI_MODEL`
- `OLLAMA_MODEL`
- `AUTOPRESO_CACHE_LOG`
- `AUTOPRESO_DEBUG_LOG`

즉 autopreso는 단순 데모를 넘어서,
**로컬 개발/실행 환경에서 어느 정도 제어 가능한 도구**라는 인상을 줍니다.

## autopreso가 의존하는 도구들
리포트에서는 아래 도구들도 함께 언급됩니다.

- Excalidraw
- Moonshine
- Vercel AI SDK

이걸 보면 autopreso는 완전히 새로운 화이트보드 앱을 처음부터 다 만든다기보다,
**기존 도구와 AI 에이전트 레이어를 조합해 새로운 상호작용 방식을 만드는 프로젝트**로 보는 편이 자연스럽습니다.

## 이 도구가 흥미로운 이유
제가 이 내용을 읽고 인상적이었던 포인트는,
발표 도구가 더 이상 “예쁘게 슬라이드를 만드는 앱”에만 머물지 않는다는 점입니다.

autopreso는 오히려:
- 생각을 말로 풀고
- 그것을 구조화하고
- 화이트보드에 즉시 반영하고
- 필요하면 로컬 환경에서도 돌리고
- 에이전트 모델도 바꿔 끼우는

방식으로 갑니다.

즉 완성된 자료를 보여주는 도구라기보다,
**설명과 사고 과정을 시각화하는 인터페이스**에 더 가깝습니다.

## 아직 헷갈리는 것
리포트만 읽고 아직 더 확인해 보고 싶은 부분도 있습니다.

- 실제 실시간 반응 속도가 어느 정도인지
- 한국어 음성 인식 품질은 어떤지
- 복잡한 구조 설명에서 화이트보드 배치가 얼마나 안정적인지
- 발표 중 수정이 계속 들어와도 맥락 유지가 잘 되는지
- 로컬 모델 조합에서 체감 품질이 어느 정도인지

이 부분은 실제 사용 경험이 있어야 더 정확히 판단할 수 있을 것 같습니다.

**발표와 설명을 입력 장치 기준으로 다시 설계하려는 시도**라고 느꼈습니다.

예전에는:
- 사람이 직접 도형을 배치하고

이제는 점점:
- 말로 설명하고

autopreso가 완성형 제품인지와 별개로,
